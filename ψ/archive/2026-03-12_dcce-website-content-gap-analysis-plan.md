# Plan — DCCE Website Content Gap Analysis vs NCAIF Sitemap

## Objective

Run a structured content gap analysis between the current DCCE website (https://www.dcce.go.th/) and the NCAIF sitemap vNext, so we can identify missing, weak, or misaligned content and propose Phase‑1 bridging actions.

## Reference model (target)

Use the NCAIF vNext sitemap as the **target structure**:

- [`ψ/incubate/DCCE/CRDB/National Climate Adaptation Information Framework.md`](National%20Climate%20Adaptation%20Information%20Framework.md)
- [`plans/2026-03-12_ncaif-sitemap-vnext-plan.md`](plans/2026-03-12_ncaif-sitemap-vnext-plan.md)

## Scope boundaries

- Focus on **content and information architecture**, not UI styling or technical infrastructure.
- Phase‑1 lens: compare only what is relevant to the interim report and the first‑phase conceptual architecture.

---

## Work plan

### 1) Capture the current DCCE site map (as‑is)

Deliverable: **DCCE site inventory table**

Fields:
- Page/section name
- URL
- Purpose/summary
- Primary audience

Method:
- Manually walk the top navigation and sub‑menus on https://www.dcce.go.th/.
- Add major content clusters (news, policy pages, projects, risk maps, portals, catalogs, downloads).

### 2) Build the NCAIF × DCCE mapping matrix

Deliverable: **Gap matrix**

Rows = NCAIF sections and sub‑sections (from sitemap vNext).

Columns:

- NCAIF section/subsection
- Intended audience
- Standards anchor (IPCC/WMO/UNFCCC/ISO element)
- Matching DCCE page(s)
- Coverage rating (None / Weak / Partial / Strong)
- Notes on mismatch

### 3) Identify gap types

Deliverable: **Gap classification list**

Check and tag:

- Missing sections (no content at all)
- Under‑specification (topic exists but lacks guidance, metadata, or framing)
- Misplaced content (exists but under wrong section)
- Fragmentation/duplication (multiple pages covering same topic without coherence)

### 4) User‑journey validation

Deliverable: **Journey gaps summary**

Test three journeys against the current site:

- Policy maker
- Technical analyst
- Local implementer

For each journey, mark which NCAIF steps are blocked, weak, or missing.

### 5) Prioritize gaps into Phase‑1 actions

Deliverable: **Phase‑1 alignment backlog**

Buckets:

1. Quick IA fixes (re‑label, re‑group existing pages)
2. New minimal content pages (e.g., loss & damage explainer, adaptation cycle overview)
3. Strategic rewiring (homepage entry points, policy‑maker access block)

### 6) Report‑ready summary

Deliverable: **Interim report insert**

- 1 page: “Current DCCE website vs NCAIF target — key gaps and priorities.”
- Include a short table: Gap → Impact → Proposed fix → Priority.
