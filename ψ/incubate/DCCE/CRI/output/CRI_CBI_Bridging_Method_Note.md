# Bridging CRI Capacity Tagging Dictionary and CBI

## 1. Purpose

This note explains how the **CRI Phase 2 capacity tagging dictionary** and the **Capacity-Based Index (CBI)** relate, and how CBI is being integrated without collapsing the SES + governance framing of CRI into a simple composite ranking.

It connects:
- The original dictionary v1.1: [`ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary.md`](ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary.md)
- The CBI method reconstruction: [`ψ/incubate/DCCE/CRI/output/CRI_CBI_method_reconstruction.md`](ψ/incubate/DCCE/CRI/output/CRI_CBI_method_reconstruction.md)
- The CBI indicator crosswalk: [`ψ/incubate/DCCE/CRI/output/CRI_CBI_indicator_crosswalk.md`](ψ/incubate/DCCE/CRI/output/CRI_CBI_indicator_crosswalk.md)
- The v2 CBI-integrated dictionary: [`ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary_v2_CBI.md`](ψ/incubate/DCCE/CRI/output/CRI_Capacity_Tagging_Dictionary_v2_CBI.md)

## 2. Conceptual spine vs Baseline index

- **CRI Phase 2** (per [`CRI Phase 2 Methodology.md`](ψ/incubate/DCCE/CRI/output/CRI%20Phase%202%20Methodology.md)) defines:
  - Capacity categories (Coping, Adaptive, Transformative) as **process- and governance-oriented** constructs under a SES lens.
  - A **two-speed measurement** regime (Baseline proxies vs Target process metrics).
  - A requirement to report **data-richness/confidence (0–3)** and to avoid pure rankings.

- **The tagging dictionary v1.1** encodes this via conceptual indicators (plan revision cycles, coordination, finance, policy integration, participation, data governance, etc.) with Baseline/Target fields.

- **CBI** contributes:
  - A politically salient, 22-indicator provincial capacity baseline (CA1–TP3) structured as three **pillars** (Coping, Adaptive, Transformative) × two **dimensions** (Asset, Process).
  - Indicator metadata grounded in Thai admin systems.
  - In CRI terms, these three CBI pillars map directly onto the three CRI capacity categories; this note uses "pillar" only when explicitly referring to CBI’s own structure.

In this design, **CRI’s methodology and dictionary remain the conceptual spine**. CBI is treated as a structured **Baseline layer** that populates some conceptual indicators with concrete, DCCE-endorsed measures.

## 3. Integration principles

1. **Asymmetric integration:**
   - CBI is integrated into CRI; CRI is not reduced to the CBI formula.

2. **Type A/B/C classification (see crosswalk file):**
   - A – CBI indicator used as a Baseline proxy for an existing concept.
   - B – CBI indicator defines a new concept in the dictionary.
   - C – CBI indicator is contextual only and excluded from the core index.

3. **Two-speed preserved:**
   - For every CBI indicator adopted, a Target metric is specified in the dictionary that moves toward process quality (e.g., disbursement timeliness, enforcement, event-log based measures).

4. **Data-richness made explicit:**
   - CBI-based Baseline proxies receive data-richness scores and limitations are documented, consistent with v1.1 rules.

5. **Profile-first outputs:**
   - CRI capacity profiles use CBI-based Baseline scores internally, but outputs remain profile + gap report + evidence spine; CBI’s composite scores are contextual, not the main product.

## 4. Examples

- **Emergency planning and drills:**
  - CP1/CP2 from CBI provide Baseline for plan existence and drill frequency.  
  - The dictionary v2 defines Target metrics on revision cadence and after-action learning quality.

- **Green buffers and transformative assets:**
  - TA1, TA2, TA3 become new concepts (Type B) for ecosystem buffers, resilience investment, and resilient infrastructure, with future Target metrics tied to engineering and ecological performance.

- **Disaster insurance:**
  - AA3 is treated as contextual (Type C) to avoid over-weighting wealth and financial-market structure in a governance-focused capacity index.

## 5. Implementation guidance

- When tagging indicators for CRI Phase 2 using v2:
  - Clearly mark where CBI codes are used as Baseline and keep Target definitions visible.  
  - Always include data-richness/confidence scores in analysis and visualization.  
  - Distinguish between **CRI profiles** (capacity + impact context) and **CBI scores** in communication with DCCE and provinces.

This bridge ensures that Tik’s CBI package is fully leveraged as a structured baseline, while CRI Phase 2 retains its SES-aligned, governance-focused methodological integrity.
