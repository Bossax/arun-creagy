# Interim Report Insert — DCCE Website vs NCAIF vNext (Key Gaps & Priorities)

## Purpose

Summarize the most critical content gaps between the current DCCE website and the NCAIF vNext target model, with Phase‑1 actions that can be executed without a full platform build.

## Key gap table (ready for report)

| Gap | Impact | Proposed fix (Phase‑1) | Priority |
|---|---|---|---|
| No Policy Maker Center landing area | Leadership lacks a decision‑ready front door | Add a policy‑maker entry block on homepage + executive summary page | Now |
| No adaptation‑cycle narrative | Users cannot follow climate → risk → action story | Create a single “Adaptation Cycle Overview” hub linking existing content | Now |
| Weak area profiles | Local officials cannot access usable risk summaries | Build provincial/area profile templates + briefing pack downloads | Now |
| Loss & Damage not explicit | Post‑event logic is invisible | Add Loss & Damage explainer + post‑event assessment page | Now |
| Measures library missing | Implementation guidance is unclear | Create Adaptation Measures Library (curated options + case links) | Soon |
| Tool fragmentation | Analysts must jump across subdomains | Create unified Tools & Data entry hub | Soon |
| Methods/standards buried in PDFs | Misuse risk and low trust | Create Methods & Standards guidance pages | Soon |

## Narrative summary (1‑paragraph)

The current DCCE website is rich in documents and specialized portals but remains **fragmented and administration‑first**. The NCAIF vNext requires a **policy‑maker front door**, a visible **adaptation‑cycle backbone**, and **area‑based briefing packs** as primary outputs. Phase‑1 improvements should therefore focus on **navigation re‑grouping and minimal new content pages** (policy‑maker entry, adaptation‑cycle overview, loss‑and‑damage explainer, briefing packs) while consolidating tool discovery into a single hub. These actions align the existing site with the NCAIF concept without requiring immediate platform rebuild.
