## Note on MVD Interpretation (Deliverable 6)
Following established project architecture (Track C), the **Minimum Viable Dataset (MVD)** is formally interpreted as a **Logical Data Model (LDM)** for the Loss & Damage domain. 
- **Beyond Flat Tables**: To avoid "technically meaningless" flat lists, the LDM provides a structured relational schema (Star Schema) aligned with the **Sendai Framework Targets A-D**.
- **The Disaster Card**: The fact table (Disaster Card) acts as the central transaction point, linking discrete **Hazard Events** to **Exposed Assets** and **Observed Impacts**.
- **Referential Integrity**: The model ensures a logical bridge between scientific climatic impact-drivers and socio-economic loss data via normalized entity relationships.
- **Goal**: This framing ensures that Deliverable 6 provides a scalable data architecture and "PDNA-ready" reporting templates rather than static, unlinked spreadsheets.
