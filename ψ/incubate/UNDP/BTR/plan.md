## Objectives (BTR)

1) **Deliver Task 4-ready content**: a complete, MPG-aligned Adaptation Chapter draft (and revision-ready structure).
2) **Task 2 enablement**: validated scope + datasets + platform requirements to support risk mapping outputs.
3) **Task 3 enablement**: a baseline framework + Data Reference Sheet concept that is feasible with Thai data flows.

## Key outputs (structured)

### Output A — Adaptation Chapter (Task 4)

- MPG-aligned outline with evidence map (what evidence supports each subsection)
- Draft narrative + tables/figures list
- Review loop plan (DCCE/UNDP validation checkpoints)

### Output B — Risk Map Platform scope pack (Task 2)

- Feature list (MVP vs nice-to-have)
- Dataset inventory + access constraints
- Integration plan (APIs, update cadence, ownership)

### Output C — Adaptation M&E baseline framework pack (Task 3)

- Indicator shortlist + definitions + data sources
- Baseline logic (how baseline values are computed and refreshed)
- Data Reference Sheet template (fields, examples, workflow)

## Dependencies (explicit)

- UNFCCC MPGs (Decision 18/CMA.1) adaptation reporting elements (must be mapped to the chapter structure).
- Thailand reference set: NAP (latest), NDC, prior NC/BTR/BUR, sector plans, and any official impact/loss datasets.
- Data custodians and permissions for hazard/exposure/impact layers (for Task 2).
- Agreement on indicator governance and update cadence (for Task 3).

## Immediate priorities (next 1–2 weeks)

1) **Lock the chapter skeleton**
   - Draft MPG-aligned outline and crosswalk table (MPG element → section → evidence sources).
   - Use [inbox/active/](inbox/active/) to validate which frameworks (e.g., FGCR, GCF/AF results frameworks) will be explicitly referenced.

2) **Decide the “indicator spine” for Task 3**
   - From [inbox/active/](inbox/active/) and [inbox/writing_notes/](inbox/writing_notes/), extract:
     - candidate indicator families (output/outcome/impact)
     - baseline approach candidates
     - data availability constraints
   - Produce a shortlist + selection criteria (SMART + feasibility + reporting usefulness).

3) **Task 2 scope validation (MVP)**
   - Confirm minimum viable platform features and dataset readiness.
   - Align with any existing platform context in [ADPC dataset review.md](<ADPC dataset review.md>) and [BTR Proposal_Task 2-Climate Risk Map Platform.md](<BTR Proposal_Task 2-Climate Risk Map Platform.md>).
