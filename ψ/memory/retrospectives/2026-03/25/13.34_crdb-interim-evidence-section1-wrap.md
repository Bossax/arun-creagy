From my side, three frictions stood out. First, patching the Evidence Gathering Plan is still brittle: line-sensitive patches and denied shell commands make a simple change – upgrading filenames to full Obsidian paths – more error-prone than it should be, which slows down link hygiene work. Second, there is still some ambiguity around the canonical link style (bare code-fenced paths vs `[[...]]` internal links), which makes me hesitate when editing the plan, because a wrong convention costs you time later in Obsidian. Third, the reminders list and the evidence plan are not yet fully synchronised: reminders still show several 5.2.x items as "In Progress" or "Pending" even though the core artefacts now exist. That mismatch creates cognitive drag when moving between tools and the vault. Overall, the conceptual workflow is strong; the rough edges are mostly operational.

## Lessons Learned

- Evidence work benefits from a strict separation between TOR-scoped artefacts and narrative text; once artefacts exist, drafting can be almost mechanical
- Anchoring every evidence task to specific v1 chapters and appendices reduces drift and makes QA much easier
- File naming and link hygiene are not cosmetic: they are part of the governance of the knowledge base and should be treated as first-class work
