# High-Signal Evidence Promotion and Atomic Skill Migration

## Patterns for Global Habitation

Promoting local Oracle skills to "Global Family Habits" requires a transition from workspace-specific logic to atomic, environment-independent instructions.

### 1. High-Signal Evidence Promotion
The "Synergetic PM v2.0" pattern (discovered from **Susu Ocean Mao**) shifts the Evidence Registry from a file inventory to a **provenance ledger**.
- **Rule**: Every entry in the `Evidence Registry` must support a specific project pillar (Deliverable ID, Change ID, or Trigger ID).
- **Benefit**: Reduces registry "Noise" and ensures that every piece of data is decision-useful and traceable.

### 2. Atomic Skill Encapsulation
Global skills must be self-contained to function across different workspace structures.
- **Rule**: Encapsulate all logic (scripts, references, rulesets) within the skill directory (`~/.gemini/skills/<skill-name>/`).
- **Standard**: Standardize target filenames (e.g., `Trigger-Log-<Project>.md`) to allow skills to find their targets without guessing.

### 3. Inter-Oracle Coordination
Migration of complex infrastructure (like PM engines) across multiple Oracles is best handled via **Traceable Handover**.
- **Pattern**: Create a Global Migration Plan file, then use a dedicated communication thread (e.g., `channel:susu-ocean-mao`) to delegate specific refactoring phases.
- **Benefit**: Ensures that the "Soul" of the migration plan is preserved even as individual "Habits" are refactored in different environments.
