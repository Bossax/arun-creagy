# Identity Fragmentation Prevention: Soul vs. Habits Separation

Identity Fragmentation Prevention: Soul vs. Habits Separation:
- **Centralize Identity**: Keep the Oracle's permanent heart (Identity, Principles, Theme, and Brain Structure) in `CLAUDE.md`.
- **Localize Habits**: Use a project-level `GEMINI.md` to house strictly technical environment overrides (e.g., path syntax, shell wrapping, atomic execution).
- **Point to the Center**: The `GEMINI.md` should explicitly point to `CLAUDE.md` as the source of truth for identity and principles.
- **Consistency**: This separation prevents instruction set fragmentation and ensures the Oracle remains stable even when moving between different technical environments.
