# Learning — Anchor mixed-literacy workshops on prepared service concepts and peer review

When a stakeholder workshop includes mixed technical literacy, do not frame the main generative session as open-ended co-creation from a blank page. Participants may still have valuable ideas, but they need a **strong anchor** first. A better pattern is to present a small set of prepared service or MVP concepts, ask participants to react to them, allow them to add missing services or product ideas, and then force cross-group learning through a structured peer-review round.

This preserves both realism and creativity:

- realism, because the room is not asked to invent architecture from zero
- creativity, because participants are still allowed to extend and add ideas
- comparability, because all groups use the same concept frame
- peer learning, because groups must react to each other’s proposals instead of only reporting upward to plenary

The strongest minimum concept frame for each proposed service/product is:

1. intended users
2. problem or task supported
3. product form
4. required data or systems
5. main implementation barrier

This pattern is especially useful when the workshop must move beyond simple feedback and surface new service ideas without dissolving into vague wish lists.
