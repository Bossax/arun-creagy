# Learning: Modular Spatial Pipelines for High-Resolution Indices

**Date**: 2026-04-20
**Concept**: Architectural Decoupling / Medallion Pipeline
**Context**: CRI Phase 1 (Stage 3) - Thailand Spatial Disaggregation

## Pattern: The "Monolithic Spatial" Trap
In early research stages, there is a tendency to bundle API ingestion, data cleaning, and heavy spatial computation (raster zonal stats) into a single "God Script." While efficient for small tests, this pattern is highly brittle for high-resolution (100m) country-wide analysis.

### Failure Signals:
- **API Drift**: External 404s or timeouts crash the entire analysis loop.
- **Compute Exhaustion**: Memory limits are hit during raster math, losing all progress from preceding download/clean steps.
- **Verification Black-Box**: Errors in name-mapping (e.g., "Bangkok" vs "Krung Thep") are hard to debug when buried inside a 500-line script.

## The Modular Solution (Medallion Spatial)
Transitioning to a decoupled pipeline where each stage is independent:

1.  **Ingestion (Bronze)**: Pure programmatic fetch. Result: Immutable raw GeoTIFFs and GeoJSONs.
2.  **Pre-processing (Silver)**: CRS alignment and Metadata Bridge creation (e.g., `standardization_mapping.csv`). Result: Harmonized join surfaces.
3.  **Calculation (Gold)**: Pure math and Zonal Statistics. Result: Risk Ratio CSVs.
4.  **Synthesis (Viz)**: Generation of maps and charts.

## Key Insights for Climate Risk Work
- **Sparsity is the Rationale**: Administrative data with high "zero" counts (e.g., 99% zero deaths) is the empirical justification for high-resolution spatial proxies (WorldPop/ESA), not a sign of data failure.
- **Metadata > Code**: Standardization of geographic naming variants should exist in a transparent CSV bridge, ensuring that the lineage of "Knowledge Correction" is preserved for policy auditors.
- **Scale Mismatch Resolution**: Provincial relief funds can only be disaggregated using "Asset-Based weights" (Building Area/Infrastructure) rather than just population, to avoid over-weighting residential vs. industrial risk.

## Oracle Philosophy Alignment
- **Nothing is Deleted**: By moving transformation logic to a `data-lineage.md`, we preserve the decision history that is usually lost in abandoned scripts.
- **Patterns Over Intentions**: Focus on the substantive identifiers (`เขต`/`แขวง`) in the raw data rather than ad-hoc IDs generated by intermediaries (chatbots/legacy ETL).
