# Learning — Afternoon workshop: capture website expectations, not governance

Learning — Afternoon workshop: capture website expectations, not governance

Pattern:
When a workshop session is framed around a website/information platform, it is tempting to turn it into governance decision capture (owners, rails, endorsement rights). That fails when participants do not have authority to decide governance, or when the sponsor explicitly wants governance handled internally.

In that case, convert the session into expectations + needs capture that still yields implementation-ready outputs.

How to implement:
1) Keep stance as Validate + Guided Feedback (show prepared structure, then let users correct/extend).
2) Replace “decision prompts” with expectation prompts that map to: top user tasks (jobs-to-do); missing pages/content blocks; trust signals required on pages (last updated, source, limitations, contact); wording clarity/glossary backlog.
3) Ensure at least one prompt captures known open choices in the IA (e.g., province profile headline framing: determinants vs impacts/L&D).
4) Capture outputs in a simple needs log format so DCCE can translate them into backlog and governance actions later.

Why this matters:
Preserves workshop realism (no fake decisions) while producing structured inputs that directly improve website structure, content templates, and slide design.
